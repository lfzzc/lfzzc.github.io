node

  • 是一个js的运行环境

    nvm

  • node version manager
  • nvm是node版本管理工具
  • 因为node版本比较多，很多时候我们可依赖多个版本，要求我们可以自由切换

    配置：

    1. 将nvm文件放到一个全英文的路径下
    2. 修改setting.txt，将里边的路径改为f:pro\zzc\nvm 和f:pro\nvm\nodejs
    3. 因为我们需要在任何目录下都可以访问nvm和node,所以需要增加全局环境变量，修改环境变量：新建NVM_HOME：值为nvm路径（f:pro\zzc\nvm），新建NVM_SYMLINK，值为f:pro\nvm\nodejs

      nvm命令：

    • nvm list 查看所有node版本
    • nvm install 6.9.1 安装node版本，如果是32位，需要在后边加上32
    • nvm uninstall 6.9.1 卸载node版本
    • nvm use 6.9.1 使用6.9.1 版本的node

      npm

  • node package manager
  • node 的包管理工具
  • npm管理包非常方便，我们只需要使用哪一个包即可

    npm命令：

    • npm install 安装一个包到本地
      例1:
      1. 新建一个npm-demo的文件夹（假设这个就是我们要做的一个网站项目）
      2. 在该文件夹下执行命令npm install bootstrap ，安装完成之后，项目根目录会多一个node_modules的文件，bootstrap就在这个文件夹下
      3. 在根目录下新建index.html，引入bootstrap,我们已经知道了我们项目依赖bootstrap,而且我们每个人的bootstrap都一样的，而且为了避免存储空间的浪费，我们只需要记录下来我们项目所依赖的包，下次我们直接安装这些包就可以了
      4. 由于需要记录项目依赖哪些东西，所以我们需要一个package.json的文件，因此需要执行命令 npm init，初始化一个 pacjage.json的文件，(备注:npm提供一些工具：gulp,我们的页面跟gulp没有关系，知识在开发阶段需要它，因此gulp属于开发阶段的依赖bootstrap，和开发，生产阶段都没有什么关系，只要我们的网页要运行就依赖它，我们的页面用到了它，bootstrap在任何阶段都需要它)
    • npm install bootstrap --save 将包的名字和版本记录在dependencies（普通项目依赖）中
    • npm install gulp --save-dev 将包的名字和版本记录在devdependencies（开发阶段依赖）中
    • npm uninstall gulp --save-dev 卸载
    • npm uninstall gulp --save 卸载
    • npn install xxx -g 全局安装 我们的包需要全局使用

      bower

  • node的应用程序依赖项管理工具
    安装：
  • npm insatall -g bower (默认全局安装会跑到C盘的appdata中，所以我们要修改npm的全局路径，在C盘用户目录下创建.npmrc文件，向其中写入cache=F:\nvm\nvmw\npm-cache prefix=F:\nvm\nvmw\npm 这样以后通过npm install -g安装的包都会安装到F盘的nvmw中)

    bower命令:

    bower init ,同npm init
    bower install xxx ,同npm install
    其它命令不再赘述.
