'前端工具02'

发表于 2016-11-04   |  

复习:

npm node应用程序依赖项管理工具,即包管理工具
bower 和npm类似,都是包管理工具,但是bower下载包时会优先查看缓存,并且与官网比较,下载需要的更新

gulp :

自动化工具

gulp工作流:

1. 新建项目  
2. 初始化环境, npm init
3. 安装gulp npm install gulp --save-dev,安装bootstrap npm install bootstrap --save
4. 新建gulpfile.js:gulp是通过gulpfile.js来实现前端自动化的
5. 它可以做4件事情: 
    - less编译,压缩,合并  
    - js压缩,合并,混淆
    - html压缩
    - image复制
6. 常用插件:gulp-less gulp-cssnano gulp-uglify gulp-concat gulp-htmlmin browser-sync

git:

  • 源代码管理工具,即版本管理工具

命令:

  • git init :初始化环境,创建一个仓库,在新项目中使用该命令,目录中会多出一个.git文件,用于记录每次的修改,在项目根目录下新建一个.gitignore文件放入不需要被记录的文件,例如*.idea,node_modules,这写文件只在开发阶段使用所以不需要被记录,在.gitignore中的文件将不会出现在 git status中
  • git add :把一个文件加入暂存区 git add .==git add –all 把所有文件加入暂存区(除过.ignore中的)
  • git commit -m ‘hello’ :提交暂存区到仓库区
  • git branch v1:新建一个v1分支
  • git checkout v1; 切换到v1分支
  • git status 显示有变更的文件
  • git log: 显示当前分支的版本历史
  • git fetch [remote] 下载远程仓库的所有变动
  • git push origin -u v1 把分之一推送到远程仓库
  • git pull origin master 把master分支从远程仓库拉过来
  • git reset –hard d25635 回退版本

github:

  • git的一个提供商
    git托管页面:git checkout -b gh-pages 新建一个gh-pages页面并切换到该分支,然后推送到远程仓库
    页面访问地址:https:lfzzczzy/github.io/项目名称
