Node.js

Node.js概述

  • Node.js是基于V8引擎的js运行环境，使用了事件驱动、非阻塞I/O模型，使其轻量又高效。
  • node是一个javaScript的运行环境（平台），所谓运行环境，是指js通过node在服务器端运行，node就像是javascript虚拟机，node不是一门语言，也不是一个框架。
  • 语言的能力取决于平台，并不取决于语言本身，语言本身只提供定义变量，定义函数，定义类型，流程控制，循环结构之类的操作
  • 对于js来说，我们常说的js是Es，大部分能力是浏览器的执行引擎决定的，例如，BOM和DOM是可以说是浏览器为我们开放出来的接口，Cordova中提供js调用摄像头，操作本地API。

    Node的实现

  • Node 内部采用 Google Chrome 的 V8 引擎，作为 JavaScript 语言解释器；
  • 通过自行开发的 libuv 库，调用操作系统资源。

    Node安装

    nvm安装

  1. 下载nvm，解压到全英文路径
  2. 编辑setting.txt（若没有则新建），
    • root配置为当前nvm.exe所在目录
    • path配置为node快捷方式所在目录
    • arch为当前操作系统位数64/32
    • proxy不用配置
  3. 配置环境变量(用户)
    • NVM_HOME=当前nvm.exe所在目录
    • NVM_SYMLINK=node快捷方式所在目录
    • PATH+=%NVM_HOME%;%NVM_SYMLINK%;
    • 打开CMD，通过set命令查看环境变量是否配置成功，set NVM_HOME,set NVM_SYMLINK,set PATH
  4. npm的目录之后再配置

    windows常用命令行操作

  • cd mkdir dir(ls) cls(clear) del(rm) 括号中的必须在新版PowerShell中使用。

    Node REPL环境

  • 全称：read eval print loop
  • 接受用户输入——>执行用户输入——>打印执行结果到控制台——>循环到下一次
  • 在cmd中输入node即可进入REPL环境，相当于chrome中的console环境。
  • 特殊变量下划线（_）表示上一个变量的返回结果
  • 通过.exit调用process.exit()退出REPL

    Node全局对象

  • global：
    类似于客户端 JavaScript 运行环境中的 window
  • process：
    用于获取当前的 Node 进程信息，一般用于获取环境变量之类的信息
  • console：
    Node 中内置的 console 模块，提供操作控制台的输入输出功能，常见使用方式与客户端类似
